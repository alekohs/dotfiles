description: Turn a vague client request into sharp clarifying questions and a crisp scope statement
---

A client/stakeholder gave me this request: $@

It's likely under-specified. Before any work:

1. Restate the request as you understand it in one sentence.
2. List the clarifying questions that actually change the implementation or estimate — ordered by how much they move scope. Skip questions whose answer wouldn't change anything.
3. For each open question, note the assumption you'd make if no answer comes, so work isn't blocked.
4. Draft a one-paragraph scope statement: what's in, what's explicitly out.

Be concise. The goal is to expose the ambiguity that would otherwise bite later, not to interrogate.
